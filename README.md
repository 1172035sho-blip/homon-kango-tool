# 訪問看護 算定判定支援ツール

訪問看護の**保険種別判定（医療保険／介護保険）と基本報酬・加算のマッチング**を支援する、ローカル完結のWebアプリです。

> **免責**：本ツールは算定支援・シミュレーション（ダブルチェック）用です。出力を公式な請求根拠として使用しないでください。最終的な算定・請求の判断と責任は、利用者（有資格者）にあります。

## 公開URL（GitHub Pages）

https://1172035sho-blip.github.io/homon-kango-tool/

どのPCのブラウザからも上記URLで利用できます。**入力したデータはそのブラウザ内（localStorage）にのみ保存され、サーバーには一切送信・保存されません。** そのため端末（ブラウザ）ごとにデータは別々です。共用PCでの利用は、ブラウザにデータが残るため避けてください。

## 使い方

### かんたん判定（登録不要・スマホ対応）

保険種別だけをその場で知りたいときは、最初に表示される「**かんたん判定**」タブを使います。

1. 上記URLを開く（またはローカルの `index.html` をダブルクリックしてブラウザで開く）
2. 訪問日と年齢（または生年月日）を入力
3. 表示される質問にタップで答える → **医療保険／介護保険／要確認** が即座に表示される

質問は回答内容に応じて必要なものだけが出ます（例：特定疾病は40〜64歳のときだけ、精神科訪問看護は65歳以上のときだけ）。40歳未満や別表7該当のように判定が確定する場合は、そこで質問が打ち切られます。

**入力内容は保存されません。** 生年月日で入力した場合のみ、その内容を利用者マスタへ登録できます（指示書の日付は未設定のまま保存されるため、あとから「利用者」タブで入力してください）。

### 月次の判定・金額まで見る場合

1. 「事業所設定」→「利用者」→「訪問記録」の順に入力
2. 「判定・計算」タブで月次の判定結果を確認（行をタップで根拠トレースを表示）
3. 「CSVエクスポート」で請求ソフトとの突合用CSVを出力

かんたん判定と「判定・計算」タブは同一の判定エンジン（`Engine.judgeInsurance`）を呼ぶため、同じ入力に対する保険種別の結論は必ず一致します。

データはすべてブラウザの `localStorage`（この端末内）に保存され、**外部には一切送信されません**。

## 最重要：報酬マスタについて

`masters.js` の既定マスタは、**金額・算定要件・有効期間がすべて `null`（未検証）** です。学習データや推測による仮の数値は入っていません。

マスタを実際の値にするには：

1. 厚労省の告示・通知PDF（令和8年度改定対応版）または請求ソフトのマスタ出力を用意する
2. PDFをClaudeとのチャットにアップロードし、マスタJSONの叩き台作成を依頼する
3. 出来上がったJSONを**必ず一次資料と照合**し、各項目の `verified` を `true` にする
4. 「報酬マスタ」タブの「JSONインポート」で取り込む

`verified: false` の項目や金額 `null` の項目は、判定結果で「要確認」として表示され、**合計金額には含まれません**。

- 参考一次情報源：厚生労働省「令和8年度診療報酬改定について」 https://www.mhlw.go.jp/stf/newpage_67729.html
- 各マスタ項目は `effective_from`（有効開始日）を持ち、改定日をまたぐ訪問には該当バージョンのみが適用されます

## 設計方針

- **決定的なルールベース**：判定・計算はテーブル参照と条件分岐のみ。LLM等による生成的判断は行いません（再現性・監査可能性のため）
- **不明値は「要確認」**：判定に必要な情報が欠けている場合、自動判定せず要確認フラグを立てます（SOAPの「記載なし」と同じ思想）
- **個人情報**：利用者は匿名ID（P0001等）で管理し、実名・詳細な病状は入力しない運用を前提とします

## ファイル構成

| ファイル | 内容 |
|---|---|
| `index.html` | 画面本体（ブラウザで開く） |
| `app.js` | 判定エンジン（保険種別判定・マスタ突合）とUI |
| `masters.js` | 報酬・加算マスタの既定値（全項目 null・未検証） |
| `tests.js` | 架空シナリオのテストスイート（架空の金額を持つテスト専用マスタを使用） |
| `styles.css` | スタイル |

## テスト

「テスト」タブの「テストを実行」で、シナリオA（70歳・要介護2・PT訪問→介護保険）、シナリオB（55歳ALS→医療保険優先）を含む10件の架空シナリオを自動照合します。基本報酬の計算テストには**実在しない架空の金額**のテスト専用マスタを使います。

## 開発フェーズ

- **Phase 1（現在）**：保険種別判定＋基本報酬マッチングの骨格、要確認の伝播、CSV出力、テストスイート、かんたん判定（登録不要の1画面判定）とスマホ対応
- **Phase 2（予定）**：優先加算5〜6個（緊急時訪問看護加算・特別管理加算・複数名訪問看護加算など）。一次資料PDFの取込が前提
- **Phase 3（予定）**：残りの加算、併算定不可・月上限チェックの本格運用、月次シミュレーション、算定漏れ検出
